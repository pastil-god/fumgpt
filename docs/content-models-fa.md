# مدل محتوای ساده برای Contentful

این پروژه فعلاً معماری خود را عوض نمی‌کند و Contentful فقط برای محتوای قابل‌مدیریت استفاده می‌شود، نه منطق پیچیده فروش.

اصل راهنما:

- متن‌ها، بنرها، FAQ، محتوای آموزشی و توضیحات نمایشی داخل CMS باشند.
- منطق فروش، سبد، پرداخت، احراز هویت، قوانین تخفیف و مسیرهای تجاری داخل کد بمانند.
- برای هر مدل، فیلدها تا حد ممکن کم و قابل‌فهم نگه داشته شوند.

مدل‌های پیشنهادی:

1. `siteSettings`
   برای تنظیمات کلی سایت مثل برند، لوگو، متن‌های هدر/فوتر، راه‌های ارتباطی و trust badgeها.

2. `homepageSettings`
   برای هیرو، CTAها، سکشن‌های صفحه اصلی، متن‌های بازاریابی، بخش اعتماد/چرا ما، مسیر توسعه و بنر پایینی صفحه.

3. `categoryContent`
   برای label و description دسته‌ها. این مدل فقط محتوای نمایشی را نگه می‌دارد.

4. `product`
   برای محتوای نمایشی محصول مثل عنوان، توضیح، تصویر، badge، متن تحویل و محتوای بازاریابی.
   قیمت نمایشی فاز 1 می‌تواند اینجا بماند، اما منطق پیچیده فروش نباید وارد CMS شود.

5. `academyArticle`
   برای مقالات آموزشی فاز بعدی.

6. `faqItem`
   برای سوالات متداول ساده و قابل‌ویرایش.

7. `announcementBanner`
   برای بنرهای بازاریابی، اطلاعیه‌ها و CTAهای ساده.

مرجع فنی این مدل‌ها داخل فایل زیر نگه‌داری می‌شود:

- `apps/web/lib/contentful/model-definitions.ts`
