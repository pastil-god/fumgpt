# راهنمای عملی مدیریت محتوا برای ادمین‌های FumGPT

این راهنما برای تیمی نوشته شده که می‌خواهد بدون کدنویسی، فروشگاه را هر روز به‌روزرسانی کند.

## 1. هر مدل برای چه کاری است؟

### `siteSettings`

برای موارد ثابت کل سایت:

- نام برند
- لوگو
- متن نوار بالای سایت
- شماره و ایمیل پشتیبانی
- تلگرام، اینستاگرام، واتساپ
- آدرس یا توضیح مسیر تماس
- متن فوتر
- کپی‌رایت
- آیتم‌های اعتماد
- متن و لینک دکمه ارتباط با پشتیبانی

### `homepageSettings`

برای صفحه اصلی:

- متن و CTAهای هیرو
- بنرهای صفحه اصلی
- تیتر و توضیح بخش دسته‌بندی‌ها
- تیتر و توضیح بخش محصولات منتخب
- تیتر و توضیح بخش خبرها
- متن بخش اعتماد / چرا ما
- متن و فازهای بخش مسیر توسعه
- نمایش/عدم نمایش هر بخش
- نوار پشتیبانی پایین صفحه

### `navigationItem`

برای منوی اصلی و لینک‌های فوتر.

### `product`

برای افزودن، ویرایش، غیرفعال کردن یا اولویت‌بندی محصولات.

### `newsArticle`

برای خبرها، اطلاعیه‌ها و مقاله‌های صفحه اصلی و صفحه `/news`.

## 2. چطور شماره یا ایمیل پشتیبانی را عوض کنم؟

1. وارد `siteSettings` شوید.
2. فیلدهای `supportPhone` و `supportEmail` را ویرایش کنید.
3. اگر نیاز بود، `supportCtaLabel` و `supportCtaHref` را هم تغییر دهید.
4. روی `Publish` بزنید.

نتیجه:

- هدر
- فوتر
- CTAهای پشتیبانی
- بعضی مسیرهای خرید

به‌صورت خودکار به‌روزرسانی می‌شوند.

## 3. چطور یک محصول جدید اضافه کنم؟

1. یک entry جدید از نوع `product` بسازید.
2. این فیلدها را حتماً پر کنید:
   - `title`
   - `slug`
   - `shortDescription`
   - `description`
   - `category`
   - `brand`
   - `price`
   - `delivery`
   - `status`
3. اگر تخفیف دارید، `comparePrice` را هم پر کنید.
4. برای تصویر اصلی، `image` را آپلود کنید.
5. اگر چند تصویر دارید، `galleryImages` را هم اضافه کنید.
6. اگر می‌خواهید در صفحه اول دیده شود، `isFeatured` را فعال کنید.
7. اگر می‌خواهید بالاتر از بقیه محصولات دیده شود، `priority` بزرگ‌تر بدهید.
8. در پایان `Publish` کنید.

نکته:

- فقط محصولاتی که `status = active` دارند در سایت عمومی نمایش داده می‌شوند.

## 4. چطور یک خبر یا مقاله جدید منتشر کنم؟

1. یک entry جدید از نوع `newsArticle` بسازید.
2. این فیلدها را حتماً پر کنید:
   - `title`
   - `slug`
   - `summary`
   - `body`
   - `publishedAt`
   - `status`
3. اگر می‌خواهید خبر در صفحه اصلی بیشتر دیده شود، `isFeatured` را روشن کنید.
4. اگر لازم بود تصویر یا ویدئو اضافه کنید.
5. در پایان `Publish` کنید.

نکته:

- فقط خبرهایی که `status = active` دارند در سایت عمومی نمایش داده می‌شوند.

## 5. چطور متن هیرو یا بخش‌های بازاریابی صفحه اصلی را عوض کنم؟

1. وارد `homepageSettings` شوید.
2. برای هیرو این فیلدها را تغییر دهید:
   - `heroTitleLead`
   - `heroTitleHighlight`
   - `heroTitleTail`
   - `heroDescription`
   - `heroPrimaryCtaLabel`
   - `heroPrimaryCtaHref`
   - `heroSecondaryCtaLabel`
   - `heroSecondaryCtaHref`
3. برای کارت اعتماد و مسیر توسعه این فیلدها را تغییر دهید:
   - `trustTitle`
   - `trustPoints`
   - `roadmapTitle`
   - `roadmapPhase1Title`
   - `roadmapPhase1Description`
   - `roadmapPhase2Title`
   - `roadmapPhase2Description`
   - `roadmapPhase3Title`
   - `roadmapPhase3Description`
4. برای بنرهای صفحه اصلی این فیلدها را تغییر دهید:
   - `heroMarketTitle`
   - `heroMarketDescription`
   - `announcementTitle`
   - `announcementDescription`
5. اگر می‌خواهید بخشی موقتاً پنهان شود، از فیلدهای `show...` استفاده کنید.
6. روی `Publish` بزنید.

## 6. اگر محتوا Publish نشده باشد چه می‌شود؟

- entry ذخیره می‌شود ولی در سایت عمومی نمایش داده نمی‌شود.
- این رفتار برای جلوگیری از انتشار ناخواسته است.
- اگر تغییری در سایت ندیدید، اول بررسی کنید که entry واقعاً Publish شده باشد.

## 7. fallback mode یعنی چه؟

fallback mode یعنی:

- یا Contentful هنوز به پروژه وصل نشده،
- یا اتصال CMS خطا داده،
- یا متغیرهای محیطی CMS تنظیم نشده‌اند.

در این حالت:

- سایت از کار نمی‌افتد
- داده‌های نمونه داخلی نمایش داده می‌شوند
- تغییرات جدید Contentful تا رفع اتصال در سایت دیده نمی‌شوند

## 8. تقسیم کار چند ادمین به چه شکل باشد؟

### Super Admin

مسئول:

- `siteSettings`
- `homepageSettings`
- تنظیمات کلی برند
- کنترل نهایی انتشار
- بررسی fallback / اتصال CMS

### Product Manager

مسئول:

- همه entryهای `product`
- قیمت، تخفیف، اولویت نمایش
- تصاویر و ویدئوهای محصول
- فعال/غیرفعال کردن محصولات

### Content Editor

مسئول:

- همه entryهای `newsArticle`
- خلاصه و متن خبرها
- تاریخ انتشار
- تصویر و محتوای رسانه‌ای خبر

### Marketing / Support Editor

مسئول:

- متن نوار بالای سایت
- اطلاعات تماس
- CTAهای پشتیبانی
- بنرهای بازاریابی صفحه اصلی
- آیتم‌های اعتماد و متن فوتر

## 9. قانون‌های ساده برای جلوگیری از اشتباه

- اسلاگ‌ها را انگلیسی و کوتاه بنویسید.
- در `siteSettings` و `homepageSettings` فقط یک entry اصلی داشته باشید.
- قبل از Publish، حتماً لینک CTAها را تست کنید.
- اگر لینک خارجی است، کامل با `https://` وارد کنید.
- برای محصولات مهم، `priority` بزرگ‌تر بدهید.
- برای خبرهای مهم، علاوه بر `priority`، گزینه `isFeatured` را هم روشن کنید.
- محصول یا خبر حذف‌شده را بهتر است اول روی `draft` ببرید، بعد حذف کنید.

## 10. چک‌لیست سریع روزانه

1. خبر جدیدی هست؟ در `newsArticle` ثبت و Publish کنید.
2. قیمت یا موجودی ویترین تغییر کرده؟ در `product` اصلاح کنید.
3. بنر کمپین یا CTA جدید دارید؟ در `homepageSettings` عوض کنید.
4. شماره، تلگرام یا متن فوتر تغییر کرده؟ در `siteSettings` اصلاح کنید.
5. اگر منو تغییر کرده، entryهای `navigationItem` را به‌روزرسانی کنید.
